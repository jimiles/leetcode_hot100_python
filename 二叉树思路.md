# Hot100 二叉树题：前置基础与逐题思路

二叉树题不适合一题一题硬记，更适合先建立一些固定共识，再看每道题调用了哪一种基础能力。

## 一、前置基础 / 共识

### 1. 二叉树题的本质通常是“遍历”

绝大多数二叉树题，都可以先从“怎么访问节点”开始想。

- `DFS`：深度优先，通常用递归
- `BFS`：广度优先，通常用队列

第一反应通常是：

- 如果题目强调“层”“每层”“从左到右”“右视图”，优先想 `BFS`
- 如果题目强调“子树信息”“左右子树”“递归返回”，优先想 `DFS`

### 2. 递归函数最关键的是“返回什么”

很多二叉树题的核心，不是会不会递归，而是 `dfs(node)` 到底该返回什么。

常见返回值：

- 返回一个数值：例如深度、路径贡献、节点个数
- 返回一个布尔值：例如这棵子树是否满足条件
- 返回一个节点：例如最近公共祖先这一类题

做题时先问一句：

`dfs(node)` 应该给父节点提供什么信息？

### 3. 前序 / 中序 / 后序，本质是处理当前节点的时机不同

三个最基础的 DFS 模板：

```python
def preorder(node):
    if not node:
        return
    # 先处理自己
    preorder(node.left)
    preorder(node.right)

def inorder(node):
    if not node:
        return
    inorder(node.left)
    # 中间处理自己
    inorder(node.right)

def postorder(node):
    if not node:
        return
    postorder(node.left)
    postorder(node.right)
    # 最后处理自己
```

使用习惯：

- `前序`：适合“先处理当前节点，再把信息往下传”
- `中序`：在 `BST` 中最重要，因为它天然有序
- `后序`：最适合“先拿左右子树结果，再算当前节点”

### 4. 有些题“递归返回值”和“最终答案”不是一个东西

这是二叉树里最容易卡住的一类题。

比如：

- 返回给父节点的，只能是一条“单边路径”
- 但当前节点的真正答案，可能需要“左边 + 右边”一起参与

这类题通常会有两个部分：

- 递归返回值：负责向父节点上传信息
- 全局变量：负责维护整棵树的最优答案

典型代表：

- `二叉树的直径`
- `二叉树中的最大路径和`

### 5. 只要出现 BST，就立刻想到“中序有序”

二叉搜索树最重要的性质：

- 左子树所有节点都小于根
- 右子树所有节点都大于根
- 中序遍历结果严格递增

这个性质会直接解决：

- `验证二叉搜索树`
- `二叉搜索树中第 K 小的元素`

### 6. 只要出现“层”，就优先想到 BFS

层序遍历的固定模板：

```python
from collections import deque

def bfs(root):
    if not root:
        return []

    queue = deque([root])
    res = []

    while queue:
        size = len(queue)
        level = []
        for _ in range(size):
            node = queue.popleft()
            level.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        res.append(level)

    return res
```

关键点不是“用队列”这么简单，而是：

- 每一轮循环处理一整层
- 用 `size = len(queue)` 把这一层切出来

### 7. 构造树的题，本质是分治

凡是“根据某种序列构造二叉树”，通常都在做三件事：

- 找根节点
- 划分左子树范围
- 划分右子树范围

然后递归构造左右子树。

所以这类题的关键不是遍历，而是：

- 根怎么找
- 左右边界怎么切

### 8. 路径题要先判断：路径是不是一定从根出发

这一点会直接决定解法复杂度。

- 如果路径必须从根开始，普通 DFS 往往就够了
- 如果路径可以从任意节点开始，就可能要用前缀和或更强的状态维护

典型代表：

- `路径总和 III`

### 9. 常见的思考顺序

拿到一道二叉树题时，可以按下面这个顺序判断：

1. 是不是 `BST`
2. 是不是“构造树”
3. 是不是“按层处理”
4. 递归函数应该返回什么
5. 最终答案是不是需要左右子树组合
6. 路径是不是必须从根开始

## 二、不同题目的对应思路

### 1. 二叉树的中序遍历

调用的基础：

- 二叉树本质是遍历
- 中序模板

思路：

这是最标准的模板题。按照“左 -> 根 -> 右”的顺序递归遍历即可。重点不是难度，而是熟悉中序遍历的写法和处理当前节点的位置。

### 2. 二叉树的最大深度

调用的基础：

- DFS
- 递归函数返回值

思路：

定义 `dfs(node)` 返回“以 `node` 为根的这棵子树的最大深度”。那当前节点的结果就是：

`max(左子树深度, 右子树深度) + 1`

这是最典型的后序递归题，因为必须先知道左右子树有多深，才能算当前节点。

### 3. 翻转二叉树

调用的基础：

- DFS 遍历每个节点
- 原地修改树结构

思路：

遍历到每个节点时，交换它的左右子节点，然后递归处理左右子树。它的本质是“每访问到一个节点，都执行同样的交换操作”。

### 4. 对称二叉树

调用的基础：

- 递归函数不一定只有一个参数
- 镜像比较

思路：

这题不是单独看一棵子树，而是比较两棵子树是否镜像对称。定义 `dfs(left, right)`：

- 如果两个都空，对称
- 如果一个空一个不空，不对称
- 如果值不同，不对称
- 然后继续比较 `left.left` 和 `right.right`
- 再比较 `left.right` 和 `right.left`

所以它本质上是“双节点同步递归”。

### 5. 二叉树的直径

调用的基础：

- 递归返回值和最终答案可能不同
- 全局变量

思路：

对父节点来说，只需要知道当前节点向下的最大深度，所以 `dfs(node)` 返回“深度”。

但直径并不只是单边深度，而是“经过当前节点的左深度 + 右深度”。所以要在每个节点更新一次全局最大值。

记忆方式：

- 返回值：给父节点的单边深度
- 全局答案：当前节点处的左右深度之和

### 6. 二叉树中的最大路径和

调用的基础：

- 和直径同类
- 返回值和最终答案拆开处理
- 全局变量

思路：

这题与“二叉树的直径”非常像。

对父节点来说，当前节点只能向上返回一条路径，因此递归返回“当前节点向下延伸的单边最大贡献”。

但对当前节点自己来说，最优答案可能是：

`左贡献 + 当前值 + 右贡献`

所以也要用全局变量维护全局最大路径和。为了避免负数拖累答案，左右贡献通常会和 `0` 比较，负贡献直接舍弃。

### 7. 二叉树的层序遍历

调用的基础：

- 只要出现“层”，优先想 BFS
- 队列按层处理

思路：

标准层序模板。用队列保存当前待访问节点，每次取出一整层，把下一层节点加入队列。重点是每轮先记录 `size`，这样才能明确当前这一轮处理的是哪一层。

### 8. 二叉树的右视图

调用的基础：

- BFS 层序遍历
- 每层只取一个节点

思路：

既然题目问的是“从右边看能看到什么”，本质就是每一层最右边那个节点。因此仍然用层序遍历，只不过每层不需要全部结果，只记录最后一个节点值即可。

### 9. 将有序数组转换为二叉搜索树

调用的基础：

- 构造题本质是分治
- BST
- 平衡要求决定选中点

思路：

有序数组要构造成平衡 BST，最自然的方法就是每次取中间元素作为根节点。这样左边部分递归构造左子树，右边部分递归构造右子树。

这题的重点不是 BST 本身，而是“为了平衡，所以根要取中点”。

### 10. 验证二叉搜索树

调用的基础：

- BST 的中序遍历结果严格递增

思路：

中序遍历 BST 时，访问顺序应该越来越大。遍历过程中维护一个前驱值 `prev`，如果发现当前节点值小于等于 `prev`，说明不满足 BST 性质。

关键不是只看当前节点和左右孩子，而是看整棵树的全局有序性。

### 11. 二叉搜索树中第 K 小的元素

调用的基础：

- BST 中序有序
- 中序遍历天然有排名顺序

思路：

中序遍历访问节点的顺序就是从小到大，因此在遍历过程中计数，数到第 `k` 个节点时，当前值就是答案。

本质上是把 BST 当成一个隐式的升序序列来处理。

### 12. 从前序与中序遍历序列构造二叉树

调用的基础：

- 构造题本质是分治
- 前序负责找根
- 中序负责划分左右区间

思路：

前序遍历的第一个元素一定是当前子树的根节点。然后去中序遍历中找到这个根所在位置：

- 左边部分属于左子树
- 右边部分属于右子树

再递归构造左右子树。

这题最关键的共识是：

- 谁告诉你根是谁：前序
- 谁告诉你左右边界：中序

### 13. 二叉树展开为链表

调用的基础：

- 前序遍历顺序
- 树结构重连

思路：

展开后的链表顺序要求和前序遍历一致，所以最直接的思路就是先按前序遍历拿到所有节点，再依次把：

- `left` 置空
- `right` 指向前序下一个节点

如果追求原地写法，也可以用逆前序递归，但理解上先抓住一点：

最终链表顺序就是前序顺序。

### 14. 路径总和 III

调用的基础：

- 路径不一定从根开始
- 普通 DFS 不够
- 前缀和

思路：

因为路径可以从任意节点开始、向下延伸，所以不能只写“从根往下加”的普通递归。更高效的做法是维护根到当前节点的前缀和。

如果当前前缀和是 `curr_sum`，目标是 `targetSum`，那么只要之前出现过前缀和 `curr_sum - targetSum`，就说明中间那一段路径和为目标值。

这是“和为 K 的子数组”在树上的版本。区别只是树是 DFS 走的，所以回溯离开当前节点时，要撤销当前前缀和出现次数。

### 15. 二叉树的最近公共祖先

调用的基础：

- 后序递归
- 递归函数返回节点
- 当前节点要根据左右子树结果做判断

思路：

定义 `dfs(node)` 返回“这棵子树里找到的目标节点或最近公共祖先”。

判断逻辑：

- 如果当前节点为空，返回空
- 如果当前节点就是 `p` 或 `q`，直接返回当前节点
- 递归去左右子树查找
- 如果左右都找到了，当前节点就是最近公共祖先
- 如果只有一边找到，就把那一边返回给父节点

这题的关键是“自底向上汇总信息”，因此更适合用后序递归理解。

## 三、最后的统一归纳

这 15 道题，实际上可以收束成下面几类固定共识：

- `基础 DFS`：中序遍历、最大深度、翻转二叉树、对称二叉树
- `DFS + 全局变量`：二叉树的直径、二叉树中的最大路径和
- `BFS 层序`：二叉树的层序遍历、二叉树的右视图
- `BST 中序有序性`：验证二叉搜索树、二叉搜索树中第 K 小的元素
- `分治构造`：将有序数组转换为二叉搜索树、从前序与中序遍历序列构造二叉树
- `DFS + 特殊技巧`：二叉树展开为链表、路径总和 III、二叉树的最近公共祖先

所以真正要记住的不是 15 道题，而是这些基础共识：

- 二叉树先想遍历方式
- DFS 先想返回值
- BST 先想中序有序
- 层的问题先想 BFS
- 构造题先想分治
- 路径题先想路径限制
- 如果答案要组合左右子树，警惕“返回值”和“最终答案”不是一回事
